# Технология ментальных моделей VZOR

## Интеллектуальная система поддержки принятия решений

---

## Суть технологии

VZOR использует методологию **решётки ментальных моделей Чарли Мангера** — подход к принятию решений, который применяет легендарный инвестор и партнёр Уоррена Баффета. Суть метода: любая сложная проблема анализируется не через одну призму, а через несколько фундаментальных моделей мышления из разных дисциплин. Это позволяет увидеть скрытые связи, слепые зоны и принять решение, учитывающее реальность с множества сторон.

В платформе VZOR этот метод реализован как **автоматизированный аналитический процесс** — ИИ-агент, обученный работать с библиотекой из 99 ментальных моделей по строгому алгоритму.

---

## Библиотека: 99 ментальных моделей из 8 дисциплин

Каждая модель — это структурированный аналитический инструмент. В основе лежит академическая классификация, охватывающая ключевые области человеческого знания:

| Дисциплина | Кол-во моделей | Примеры |
|---|---|---|
| **Общие инструменты мышления** | 10 | Мышление от первых принципов, Мышление второго порядка, Инверсия, Бритва Оккама |
| **Наука** | 20 | Катализаторы, Экосистемы, Эволюция, Стимулы, Инерция, Рычаг |
| **Системное мышление** | 11 | Обратная связь, Узкие места, Запас прочности, Критическая масса, Эмерджентность |
| **Математика** | 7 | Выборка, Случайность, Регрессия к среднему, Глобальный и локальный максимумы |
| **Экономика** | 12 | Спрос и предложение, Альтернативные издержки, Монополия и конкуренция, Пузыри |
| **Искусство** | 11 | Фрейминг, Контраст, Аудитория, Нарратив, Персонаж |
| **Военная стратегия** | 5 | Асимметричная война, Война на два фронта, Разведка на передовой |
| **Человеческая природа** | 23 | Предвзятость подтверждения, Социальное доказательство, Ошибка выжившего, Влияние стресса |

### Стандартизированная структура каждой модели

Каждая из 99 моделей содержит пять обязательных разделов:

1. **Описание** — что это за модель и какую концепцию она отражает
2. **Когда избегать** — в каких ситуациях модель неприменима или опасна
3. **Ключевые слова для ситуаций** — контексты, в которых модель наиболее полезна
4. **Шаги мышления** — пошаговый алгоритм рассуждения (от 3 до 7 шагов)
5. **Коучинговые вопросы** — вопросы для углублённого применения модели к конкретной ситуации

Такая структура позволяет ИИ-агенту не просто «знать» о модели, а **последовательно применять её алгоритм** к проблеме пользователя.

---

## Как это работает: 5-шаговый аналитический процесс

Когда пользователь задаёт вопрос, агент VZOR запускает строгий аналитический протокол:

### Шаг 1. Диагностика проблемы

Агент не бросается отвечать сразу. Он **анализирует проблему** и, если она сформулирована нечётко, задаёт до 5 уточняющих вопросов. Результат фиксируется в документе `problem_diagnosis.md`.

*Пример: Пользователь спрашивает «Менять подрядчика или дожимать?» — агент уточняет: каков объём оставшихся работ? есть ли у подрядчика финансовые проблемы? каковы ваши контрактные рычаги?*

### Шаг 2. Выбор моделей (воронка отбора)

Агент проходит через **5 подшагов отбора**, чтобы выбрать максимум 3 наиболее релевантные модели:

1. **Анкета** — дополнительные вопросы пользователю для сбора контекста
2. **Первичное сканирование** — быстрый проход по всем 99 моделям, составление списка кандидатов
3. **Глубокий анализ** — чтение полного описания каждого кандидата, критическая оценка пригодности
4. **Финальный отбор** — выбор максимум 3 моделей, ранжирование по значимости
5. **Документирование** — фиксация логики отбора в `mental_models_selected.md`

*Почему максимум 3? Цель — глубина, а не поверхностный охват. Лучше применить 2-3 модели детально, чем 10 поверхностно.*

### Шаг 3. Глубокий анализ по каждой модели

Для каждой выбранной модели агент **строго следует пошаговому алгоритму** из раздела «Шаги мышления». Это не свободная интерпретация — это последовательное прохождение каждого шага с привлечением:

- Фактов и данных из базы знаний
- Внешних источников и кейсов
- Коучинговых вопросов модели

Результат каждого анализа сохраняется в отдельном файле `reasoning_<модель>.md`.

*Пример: Модель «Мышление второго порядка» для вопроса о подрядчике — агент проходит шаги: (1) определить прямое последствие замены, (2) спросить «а что потом?» — новый подрядчик тоже может сорвать сроки, (3) оценить последствие третьего порядка — потеря переговорной позиции с новым исполнителем.*

### Шаг 4. Синтез — главная ценность

Это **ключевой этап**, который отличает систему от простого списка рекомендаций. Агент:

- Читает все файлы рассуждений
- **Сплетает выводы моделей воедино** — находит, где они дополняют друг друга, где противоречат
- Выявляет **глубинные паттерны**, которые видны только при взгляде через несколько линз одновременно
- Формирует структурированный отчёт с конкретными рекомендациями

### Структура итогового отчёта

Пользователь получает документ `analysis_report.md` с чёткой структурой:

1. **Резюме** — суть проблемы и главный вывод в 2-3 предложениях
2. **Постановка проблемы** — формализованное описание ситуации
3. **Анализ по моделям** — для каждой: почему выбрана + детальный разбор
4. **Синтез и интегрированные выводы** — как модели связаны между собой, какие паттерны возникают
5. **Варианты действий и рекомендации** — конкретные шаги
6. **Источники** — ссылки на данные и кейсы

## Техническая реализация

### Архитектура

```
Пользователь (чат VZOR)
        |
        v
   VZOR Frontend --> API Gateway --> ИИ-Агент (Agent Zero)
                                          |
                                          |-- Библиотека моделей (99 файлов)
                                          |-- Векторный поиск (FAISS)
                                          +-- Рабочее пространство (файлы анализа)
```

- **99 ментальных моделей** загружены в базу знаний агента и проиндексированы через **FAISS** (векторный поиск по семантическому сходству)
- Агент находит релевантные модели через комбинацию сканирования имён файлов и семантического поиска по содержимому
- Все промежуточные файлы анализа сохраняются, что обеспечивает **полную прозрачность** процесса рассуждений

### Два режима работы

| | Глубокий режим | Быстрый режим |
|---|---|---|
| **Анкета** | Да (до 5 вопросов) | Нет |
| **Отбор моделей** | 5 подшагов | 4 подшага |
| **Глубина анализа** | Максимальная | Стандартная |
| **Применение** | Стратегические решения | Оперативные вопросы |

## Конкурентные преимущества

### 1. Системность вместо интуиции

Обычный ИИ-чатбот даёт ответ «из головы» — быстро, но поверхностно. VZOR запускает **документированный аналитический процесс** с фиксацией каждого шага. Пользователь видит не только ответ, но и **как агент пришёл к выводу**.

### 2. Междисциплинарный подход

99 моделей из 8 дисциплин позволяют смотреть на бизнес-задачу через призму физики (инерция, рычаг), биологии (экосистемы, эволюция), экономики (альтернативные издержки), психологии (когнитивные искажения) и военной стратегии (асимметричная война). Это даёт перспективы, которые невозможно получить, оставаясь в рамках одной дисциплины.

### 3. Воспроизводимость и прозрачность

Каждый анализ сохраняется как набор файлов:
- `problem_diagnosis.md` — диагностика
- `questionnaire.md` — анкета
- `mental_models_selected.md` — логика выбора моделей
- `reasoning_*.md` — детальный ход рассуждений по каждой модели
- `analysis_report.md` — итоговый отчёт

Это позволяет **вернуться к любому решению**, проверить логику, передать анализ коллеге или обновить при изменении обстоятельств.

### 4. Защита от когнитивных искажений

23 модели из категории «Человеческая природа» специально направлены на выявление типичных ошибок мышления: предвзятость подтверждения, ошибка выжившего, эффект якоря, влияние стресса. Агент не просто анализирует ситуацию — он **проверяет, не обманывает ли пользователь сам себя**.

### 5. Качество через ограничение

Система принудительно ограничивает анализ **максимум 3 моделями** за сессию. Это контринтуитивно, но именно это обеспечивает глубину: вместо поверхностного применения десятка фреймворков, агент детально проходит каждый шаг мышления 2-3 самых релевантных моделей.

## Примеры применения

### Девелопмент и строительство
- Оценка рисков при выборе подрядчиков
- Анализ решений «менять / дожимать / ждать»
- Инвестиционные решения по объектам
- Стратегия ценообразования на этапе продаж

### Бизнес-стратегия
- Выход на новый рынок
- Оценка конкурентных угроз
- Решения о партнёрствах и M&A
- Антикризисное планирование

### Операционные решения
- Оптимизация цепочки поставок
- Кадровые решения (нанять / уволить / реструктуризировать)
- Приоритизация проектов при ограниченных ресурсах

## Итог

Технология ментальных моделей VZOR — это не просто ИИ-чатбот, а **система структурированного мышления**, которая:

- Использует проверенную методологию Чарли Мангера
- Опирается на библиотеку из 99 моделей из 8 дисциплин
- Работает по строгому 5-шаговому алгоритму
- Документирует каждый шаг рассуждения
- Синтезирует выводы из разных дисциплин в единую картину
- Защищает от когнитивных искажений

Пользователь получает не мнение, а **аналитический отчёт консультанта мирового класса** — с обоснованием, альтернативами и конкретными рекомендациями.
