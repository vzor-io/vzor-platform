# VZOR v3.15 — Глубокий UX-анализ и рекомендации

> **Статус:** Черновик для согласования
> **Дата:** 2026-02-05
> **Цель:** Критический анализ платформы с точки зрения психологии, маркетинга и UX

---

## 1. ТЕКУЩЕЕ СОСТОЯНИЕ VZOR

### Что есть сейчас:
- 3D облако точек (Three.js)
- 5 категорий: Development, Finance, Real Estate, Marketing, Sales
- Drill-down в категорию → создание задач
- Node Editor для связей между задачами
- Голосовой ввод
- Тёмная тема + cyan (#00ffff) акценты

### Эмоция которую передаёт:
- **Технологичность** — sci-fi эстетика
- **Контроль** — данные как точки в пространстве
- **Холодность** — нет "человеческого" тепла

## 2. КРИТИЧЕСКИЙ АНАЛИЗ

### 2.1 Что ХОРОШО (не трогать):

| Элемент | Почему работает |
|---------|-----------------|
| 3D облако точек | Уникально, запоминается, "wow-эффект" |
| Минимализм UI | Не отвлекает от контента |
| Тёмная тема | Профессионально, современно |
| Drill-down навигация | Интуитивно понятно |

### 2.2 Что ПРОБЛЕМА:

| Проблема | Влияние | Критичность |

## 3. ПСИХОЛОГИЧЕСКИЙ АНАЛИЗ

### 3.1 Когнитивная нагрузка

**Проблема:** При входе пользователь видит:
- Облако точек (красиво, но что это?)
- 5 категорий (куда нажимать?)
- Кнопка CTA "Access VZOR Standard" (что это даст?)

**Решение:** Принцип "Progressive Disclosure" — показывать информацию постепенно

### 3.2 Эмоциональный дизайн (по Дону Норману)

| Уровень | Текущее состояние | Рекомендация |
|---------|-------------------|--------------|
| **Visceral** (первое впечатление) | Сильное ✓ | Сохранить |
| **Behavioral** (удобство) | Слабое ✗ | Улучшить feedback |
| **Reflective** (смысл) | Неясно ✗ | Добавить storytelling |

### 3.3 Принцип Fitts's Law

Текущие проблемы:
- Категории в 3D — сложно попасть мышью
- Маленькие кнопки (icon-bar 36px)
- Task points слишком мелкие

| Принцип Alto | Как применить к VZOR |

## 5. КОНКРЕТНЫЕ РЕКОМЕНДАЦИИ

### 5.1 КРИТИЧНЫЕ (исправить сначала)

#### A. Onboarding — первые 5 секунд

```
Текущее: Пользователь видит облако и не знает что делать

Решение: Добавить subtle hint при первом входе
- Полупрозрачный текст под заголовком VZOR:
  "Нажмите на категорию для начала"
- Исчезает после первого клика
- Сохраняется в localStorage (показывать только новым)
```

#### B. Task Points должны вращаться с облаком

```
Текущее: Облако крутится, task points стоят на месте (БАГ)

Решение: Добавлять task points в ту же группу (group)
что и основное облако, чтобы они вращались вместе
```

#### C. Visual feedback при действиях

```
Текущее: Создал задачу → ничего не произошло визуально

Решение:
1. Toast уведомление "Задача создана" (уже добавлено, но не работает)
2. Анимация появления точки (scale 0 → 1 с bounce)
3. Subtle pulse на новой точке 2-3 секунды
```

---

### 5.2 ВЫСОКИЙ ПРИОРИТЕТ

#### D. Hover состояния для 3D объектов

```
Текущее: Непонятно что кликабельно

Решение:
1. При hover на категорию:
   - Увеличение scale 1.0 → 1.15
   - Glow эффект
   - Cursor: pointer

2. При hover на task point:
   - Показать название задачи (tooltip)
   - Увеличение точки
   - Подсветка связей (если есть)
```

#### E. Индикация голосового ввода

```
Текущее: Нажал микрофон → ничего не видно (+ микрофон "улетает")

Решение:
1. Исправить CSS чтобы микрофон не двигался
2. Красная пульсация вокруг кнопки
3. Текст "Говорите..." появляется рядом
4. Waveform визуализация (опционально)
```

---

### 5.3 СРЕДНИЙ ПРИОРИТЕТ

#### F. Градиентный фон вместо чёрного

```
Текущее: background: #000000

Решение: Subtle радиальный градиент
background: radial-gradient(
    ellipse at center,
    rgba(10, 15, 20, 1) 0%,
    rgba(0, 0, 0, 1) 70%
);

Почему: Добавляет глубину, не отвлекает
```

#### G. Звуковой feedback (опционально)

```
Решение: Subtle звуки для ключевых действий
- Создание задачи: мягкий "pop"
- Drill-down: whoosh
- Ошибка: gentle notification

ВАЖНО: Выключены по умолчанию, включаются в Settings
```

#### H. "Focus Mode"

```
Решение: Кнопка в углу для минимального интерфейса
- Скрывает: icon-bar, header, все панели
- Оставляет: только облако + task input
- Для презентаций и глубокой работы
```

## 8. КРИТИЧЕСКАЯ ОЦЕНКА: НУЖНЫ ЛИ ИЗМЕНЕНИЯ?

### Аргументы ЗА изменения:
- Текущие баги мешают использованию
- Нет обратной связи = неуверенность пользователя
- Конкуренты (Notion, Linear) имеют отточенный UX

### Аргументы ПРОТИВ больших изменений:
- Визуальный стиль уже уникален и запоминается
- Слишком много анимаций = отвлечение
- Бизнес-пользователи ценят скорость, не красоту

### МОЙ ВЫВОД:

**Нужны ТОЧЕЧНЫЕ улучшения, не редизайн.**

Фокус на:
1. Исправить баги (это не обсуждается)
2. Добавить feedback (toast, hover, анимации)
3. Onboarding для новых пользователей

НЕ нужно:
- Менять цветовую схему
- Добавлять сложные анимации
- Копировать Alto's Odyssey дословно

## 9. ЦВЕТОВАЯ СИСТЕМА И КОНТРАСТ

### 9.1 Текущие проблемы:

| Проблема | Где | Влияние |
|----------|-----|---------|
| Слишком тёмный фон | Development drill-down | Глаза устают, не видно границ |
| Облако слишком тусклое | Все режимы | Теряется на чёрном фоне |
| Task points мелкие | При создании задач | Не видно что создалось |
| Нет контраста между элементами | Весь UI | Всё сливается |

### 9.2 Рекомендации по цвету:

```
ТЕКУЩЕЕ:
- Фон: #000000 (чистый чёрный)
- Облако: opacity ~0.3-0.5
- Task points: size 15

РЕКОМЕНДАЦИЯ:
- Фон: radial-gradient от #0a1015 (центр) к #000000 (края)
- Облако: opacity 0.5-0.7 (ярче на 30%)
- Task points: size 20-25 (больше на 50%)
- Добавить subtle grid или "туман" для глубины
```

### 9.3 Пропорции на экране:

```
Текущее распределение внимания:
┌────────────────────────────────┐
│  [много пустоты]               │
│                                │
│        ○ облако ○              │  ← 60% экрана
│           ○                    │
│                                │
│  [input внизу]                 │  ← 10% экрана
└────────────────────────────────┘

Рекомендация:
- Облако должно занимать 70-80% viewport
- UI элементы по периферии (не мешают)
- Task points крупнее и ярче чем фоновые точки
```

## 10. КОНЦЕПЦИЯ "ОБЛАКО → ЗАДАЧИ"

### Метафора:

```
ОБЛАКО ВОЗМОЖНОСТЕЙ          КОНКРЕТНЫЕ ЗАДАЧИ
     (потенциал)          →      (реальность)
         ·  ·                      ●━━━●
      · · · · ·                   ╱     ╲
     · · · · · ·          →      ●       ●
      · · · · ·                   ╲     ╱
         ·  ·                      ●━━━●
     [плотное]                 [разреженное]
```

### Механика:

1. **Начальное состояние:**
   - Облако плотное (много точек, высокая opacity)
   - Нет task points
   - Ощущение: "много возможностей"

2. **При создании задачи:**
   - Из облака "вытягивается" точка
   - Облако становится чуть менее плотным
   - Task point появляется крупнее и ярче
   - Ощущение: "конкретизирую возможности"

3. **Много задач:**
   - Облако разреженное (видно сквозь него)
   - Граф задач доминирует
   - Ощущение: "я структурировал хаос"

### Реализация:

```javascript
// При создании задачи:
function addTaskPoint(text, category) {
    // 1. Создать яркую task point (size 22, opacity 1.0)

    // 2. Уменьшить плотность облака
    cloudMaterial.uniforms.uDensity.value *= 0.97; // -3% за каждую задачу

    // 3. Анимация "вытягивания" из облака
    // Точка появляется из центра облака и летит к позиции
}
```

## 11. ВИЗУАЛЬНАЯ ИЕРАРХИЯ

### Порядок важности (что должно быть виднее):

| Приоритет | Элемент | Текущее | Рекомендация |
|-----------|---------|---------|--------------|
| 1 | Task Points (задачи) | size 15, тусклые | size 22, яркие, glow |
| 2 | Категории (labels) | opacity 0.7 | opacity 0.9, крупнее |
| 3 | Облако (фон) | opacity 0.5 | opacity 0.6, но разреживается |
| 4 | UI элементы | разбросаны | группировать, subtle |

### Размеры элементов:

```
Текущее:
- Cloud points: 2-3px
- Task points: 15px (мало отличается!)
- Category labels: ~14px font

Рекомендация:
- Cloud points: 1-2px (мельче = больше "туман")
- Task points: 20-25px (явно крупнее)
- Category labels: 16-18px font, letter-spacing
- Connections (линии): 2px, gradient opacity
```
